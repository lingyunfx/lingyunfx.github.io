废话不多说，这里为rez做个简略的介绍。

介绍

Rez是一个跨平台程序包管理工具，它可以给程序配置独立的环境。
与其它程序包环境管理不同的是，所有的程序包的安装都在一个中央统一的位置，
每个独立环境只是引用了这些包，所以它的环境配置是轻量的。

概括它的工作步骤大概就是：

  • 获取包请求
  • 解析依赖
  • 构建目标环境

一个最简单的用法：
比如你想使用maya2017和2.8.3的redshift版本，只需要输入命令：
rez-env maya2017 redshift-2.8.3

这个时候rez则会去寻找maya2017和redshift-2.8.3的软件包。（在package仓库中）
然后根据包内的package.py定义的依赖进行解析，最后完成环境构建。

环境创建好后，在终端里输入maya即可打开软件，
同时也设置好了redshift的环境与指定的版本。

要如何定义软件包，如何使用软件包这取决于你自己的需求。
在每个软件包下都有一个package.py文件。类似这样：

name = "requests"

version = "2.8.1"

authors = ["Kenneth Reitz"]

requires = [
    "python-2.7+"
]

def commands():
    env.PYTHONPATH.append("{root}/python")

这个文件描述了软件包名称，依赖项，以及会执行的命令等。
关于怎么配置包，构建包等等问题，文档里将的比较详细，这里就不赘述了。
