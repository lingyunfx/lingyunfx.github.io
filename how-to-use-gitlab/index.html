GitLab使用

zhangly 2021-11-24 13:06:01
Categories： Geek > GitLab Tags：

注册账号

https://gitlab.lingyunfx.com/

在这个链接地址注册即可，注册完后@lingyun在管理员后台进行激活账户就可以使用了。

创建一个project

在你的gitlab主页，选择Create new projec - Create blank project

  • Private：私人仓库（仅自己可见），或者明确授权给某个用户/用户组，才可以拥有访问权。
  • Internal：内部仓库，只要是gitlab.lingyunfx.com的用户都可以看到。
  • Public：公开仓库，任何人都可以看到。

绑定Pycharm

1.安装git

首先确保本机已经安装了git软件，如果没有安装在下面的链接进行下载安装。

https://git-scm.com/downloads

安装完成后，查看pycharm设置，git路径是否已经配置好。

2.生成ssh key

在桌面右键，打开git终端

输入命令ssh-keygen -t rsa -b 4096 -C "你的邮箱地址"

从输出可以看到公钥文件的保存路径（c:/User/用户名/.ssh/id_rsa.pub），用记事本打开它，复制里面内容。

登陆你的gitlab账号：

1.选择SSH Keys标签

2.粘贴刚刚复制的密钥

3.输入一个名称（比如公司电脑就写“公司电脑”，方便自己区分管理）

4.点击Add key即可

3.打开Pycharm，从Gitlab拉取项目

复制需要拉取项目的SSH地址

打开Pycharm点击Get from VCS

粘贴到URL，点击Clone：

这个时候就可以开始对代码添加和修改了。

需要提交或者更新代码，都建议用git的终端来实现。（不要用pycharm自带的git操作）

在项目文件夹右键：

在终端里进行提交

git config user.name "zhangly"                     # 设置在这个仓库中，你的用户名（本仓库设置一次即可）
git config user.email "zhangly.basefx@gmail.com"   # 设置在这个仓库中，你的邮箱（本仓库设置一次即可）
git status                                         # 查看文件修改状态
git add .                                          # 添加所有修改的文件到待提交区
git commit . -m 'hah'                              # 提交 -m 参数为本次提交的描述
git push origin main                               # 推到远程仓库，main主分支

从本地推新的仓库到远程

git config --global user.name "your_name"
git config --global user.email "your_email"

git init
git add -A
git commit -m "init blog"
git remote add origin git@gitlab.com:username/username.gitlab.io.git
git push -u origin main -f
